## Systemic Patterns (the flow story)

These recur across surfaces; fixing them once fixes many findings:

1. **Light mode is broken product-wide (8/9 surfaces).** The theme toggle ships a second theme that white-outs text and blacks-out cards. Single highest-impact systemic defect. Root cause: `globals.css:66-150` override hack vs hardcoded hex/`text-white`/canvas colors.
2. **WCAG-AA contrast fails everywhere.** `text-zinc-600/700/800` for labels, metadata, methodology, skip controls — verified ~2.5:1 on dark.
3. **Errors are swallowed (no recovery states).** `catch {}` on dashboard, screener (fetch-fail shows empty-filter state), track-record, **paper-trading financial actions** (silent failed BUY/SELL/close), email capture, settings. Trust killer on a money product.
4. **Keyboard/ARIA gaps on interactive elements.** Clickable `<article>` cards, sortable `<th onClick>`, mouse-only toggle `<div>`, fake `role=menu` with no key contract, modals with no dialog semantics/Escape/focus-trap.
5. **No single navigation source of truth.** Two desktop navbars (`navbar.tsx` vs `PageNavBar.tsx`) with 5 disagreeing link sets, a third dead `mobile-nav.tsx`, a **double footer on 10+ pages**, and `layout.tsx` renders no top nav at all.
6. **Persistent-shell overload.** `layout.tsx` fires up to 8 always-on widgets on every route; 3 collide in the same bottom-right corner (OnboardingChecklist + FeatureUnlockBanner + PWAInstallPrompt).
7. **Onboarding swarm.** 7 distinct mechanisms; 4 fire at once on first dashboard load; 3 duplicate "Getting Started" checklists; no unified skip (4 controls, 4 localStorage keys).
8. **Conversion incoherence.** No stable primary CTA on landing — A/B randomization makes the primary action AND brand color non-deterministic (variant B paints the whole site purple). Elite tier says "$99 self-serve" on /pricing but "Contact us" on /billing. `/contact-sales` is orphaned.
9. **Glassmorphism + crypto-neon as the default aesthetic**, not a deliberate choice.
10. **Dead code in the bundle:** third mobile-nav (0 importers), orphaned `alert-rule-builder.tsx` (with a `Math.random()` fake trigger), duplicate `onboarding-checklist.tsx`, dead `EliteInterestForm` import.

## Cross-Surface Flow Assessment

- **First-visit → trust:** landing throws 4+ competing CTAs and jargon (`confluence`, `TP1/TP2/TP3`, `profit factor`) before anything explains them; the one block a skeptic needs (real track record) is often empty (`proof-hero.tsx:49-86`), and "live" proof is partly synthetic (`Math.random()` signals) while the page claims "every trade verified".
- **Visitor → customer:** landing CTA randomization → /pricing (tier cards duplicated as an 11-row table at the decision point) → checkout → billing page that is dark-only and contradicts the Elite price shown moments earlier. Orphaned custom-tier funnel.
- **Sign-up → activation:** onboarding swarm hits immediately; no unified skip frustrates power users, while the duplicate checklists confuse first-timers.
- **Daily use → action:** dashboard buries the actual signals under chrome; screener misdiagnoses fetch failures as "no results"; paper-trading and settings fail silently on the financially consequential actions.
