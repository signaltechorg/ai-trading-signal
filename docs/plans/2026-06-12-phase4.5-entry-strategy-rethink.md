# Phase 4.5 — Entry-strategy rethink: find a deployable edge after costs

Date: 2026-06-12

## Why this phase exists

Phase 4 proved the current entries (classic momentum, vwap-ema-bb mean-reversion) have no deployable per-regime edge after ~0.4% round-trip crypto-perp costs on H1. This phase is a research-led search for an entry that DOES survive costs — built and validated through the Phase 4 costed regime-conditioned backtest harness (the test bench, not speculation). The outcome is genuinely uncertain; "nothing clears robustly" is a valid, business-critical result that points to a carry/cross-sectional data-acquisition pivot.

## Evidence base (two independent 2026-06-12 studies, both grounding this design)

**Deep research (cost-focused literature survey).** The governing law: viability ∝ (edge per trade) / (cost per trade), and cost is FIXED at ~0.4% round-trip regardless of timeframe. So the only levers are bigger move per trade (higher TF, wider targets) and fewer trades. Breakeven cost for intraday crypto momentum is 3–10 bps; we pay ~40 bps — H1 single-asset timing is structurally foreclosed, not a tuning problem. The one documented timing edge that survives realistic costs is **daily time-series momentum (~28-day lookback, ~5-day hold), Sharpe ~1.5 at 15 bps** (Han/Kang/Ryu 2023: "TS momentum strong, cross-sectional weak" after costs; Springer 2023). Cross-sectional momentum is weak/fragile out-of-sample. Carry/funding harvesting is the strongest raw edge (Sharpe 6+ in-sample, decaying into 2025) but is NOT a timing signal and needs funding-rate data we don't have. Target widths below ~2% gross are structurally unviable; need 4–8% multi-day moves.

**Empirical edge-map (our harness, 108 cells, 3 symbols × 3 TFs × 3 entries × geometry sweep, deterministic).** The ONLY trustworthy positive cost-adjusted cell: **classic momentum + wide 4R targets on H1, BTC +4.50%/n143/PF1.15 and ETH +2.93%/n78/PF1.12** (SOL break-even). That is +0.03–0.05%/trade after costs — real but marginal. Every entry at the live 2R geometry is negative with a trustworthy sample on every timeframe. Higher TF (H4/D1) did NOT concentrate edge — raw edge collapsed as friction shrank, and every D1 cell was thin (729 daily bars / 2yr). **Critical caveat: the map ran the existing `classic` scorer on D1 bars — NOT a proper daily time-series momentum filter. The research's #1 candidate was never actually tested, and the 2-year D1 window is too short to test it.**

**Synthesis.** Both halves converge: the edge is in fewer trades with larger edge per trade (wider targets + harder selectivity), and the 4R result is the empirical proof of the research's cost arithmetic. Two candidates are worth building and testing properly:
- (A) The empirically-confirmed-but-marginal: classic momentum + wide (≥4R) target + explicit selectivity on H1.
- (B) The research-favored-but-untested: a proper daily time-series momentum entry on adequate daily history.

## Design decisions

**D1 — Deep daily history.** Backfill BTC/ETH/SOL (and the other liquid majors we can — BNB, XRP, ADA, DOGE, AVAX, LINK, DOT) daily from the earliest available (Binance has BTC/ETH back to ~2017–2020) into the gitignored candle dir. ~2000+ daily bars per symbol vs the current 729 — enough for a real daily test with walk-forward folds and per-regime cells that aren't all thin. (Raw candle dumps NOT committed; gitignored.)

**D2 — `daily-momentum` entry module.** New entry module in `packages/strategies/src/entry/daily-momentum.ts` implementing time-series momentum: long when trailing N-day return > 0 (price above the N-day moving average, default N=28); short on the inverse; exit on momentum flip (natural ~multi-day hold). Pure, deterministic, tested. This is the research's #1, properly specified — the thing the empirical map did NOT test. Reuses existing indicator helpers; no parity-drift reimplementation.

**D3 — Harness knobs (the two levers the map flagged missing).** (a) A TP-R-multiple flag on the research CLI so wide targets (3R/4R/…) are first-class, not a scratch script. (b) A minimum-confidence / selectivity filter the backtest can apply, so "only the strongest setups" is testable on any entry (the map could only infer selectivity indirectly via the 4R target). Both additive to the harness; default path byte-identical (existing backtest tests stay green).

**D4 — Validation run (the make-or-break experiment).** Run the costed harness (crypto perp costs, walk-forward folds, deep daily history) on: (i) daily-momentum on D1 across the symbol set; (ii) classic + 4R + selectivity on H1; (iii) daily-momentum + wide targets. Report per (symbol, entry, geometry, fold): trades, win rate, cost-adjusted expectancy/return, robustness across symbols. A candidate is DEPLOYABLE only if cost-adjusted expectancy > 0 with ≥30 trades, robust across the majority of symbols (not a one-symbol fluke), and stable across folds — NOT a thin or single-symbol artifact.

**D5 — Decision + ship.** If a candidate clears D4 robustly: wire it as a real preset / `STRATEGY_PROFILE` (the honest dispatch C3 established) so it is actually deployable, register the evidence (REGISTRY + experiment JSON), and the product repositions toward the surviving timeframe/strategy. If NOTHING clears robustly: report that plainly as the finding — single-asset OHLCV timing is marginal-at-best after costs, and the honest strategic path is the carry/cross-sectional pivot (acquire Binance funding-rate data, build a carry route) rather than shipping a fragile timing signal. Either outcome is documented in the umbrella plan + a runbook/decision memo.

**D6 — Out of scope (logged).** Funding-rate carry and cross-sectional rotation (data-acquisition / separate-engine efforts — recommended in the decision memo if timing proves a dead end, but NOT built here); live activation of any surviving strategy on the Pro broadcast (still operator/shadow-gated per Phase 4); position sizing (Phase 5).

## Verification gates

- Per commit: touched jest suites green in the worktree; `npm run build:signals`/`build:all` green; no console.log in library code (CLI stdout exempt); determinism (same candles+code → identical metrics).
- daily-momentum module: unit tests for the momentum signal (known fixtures), entry/exit timing, no lookahead.
- Validation run: deterministic; thin cells (<30 trades) flagged; robustness across symbols + folds reported; NO parameter tuning to make numbers look good — report what the data says.
- The decision (D5) is evidence-bound: a strategy is promoted ONLY on a robust positive; a negative result ships as an honest verdict, not a buried failure.

## The honest framing

This phase may conclude that no single-asset OHLCV timing strategy clears costs robustly — that the marginal H1+4R cell is the ceiling and it's too fragile to sell. If so, that is the deliverable: the track-record-as-moat thesis needs the carry/cross-sectional pivot, not another entry tweak. The machinery (regime engine, costed harness, honest attribution, shadow recorder) remains the durable asset; it now has a validated method for telling edge from noise, which is what kills bad strategies before they reach a customer.
